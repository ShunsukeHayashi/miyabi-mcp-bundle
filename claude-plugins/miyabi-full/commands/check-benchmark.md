# Benchmark Implementation Check

**このコマンドを実行してベンチマーク実装前のチェックを実行します**

---

## 🔍 ベンチマーク実装チェック

### Phase 1: チェックリスト表示

まず、ベンチマーク実装チェックリストを表示します：

```bash
cat .claude/BENCHMARK_IMPLEMENTATION_CHECKLIST.md
```

### Phase 2: ユーザーへの確認事項

以下の情報を**必ず**ユーザー様に確認してください：

#### 1. 公式リポジトリ確認

```
調査対象: [ベンチマーク名]

公式リポジトリ:
- URL: [確認したURL]
- README確認: ✅/❌
- docs/確認: ✅/❌
```

#### 2. 公式ハーネス確認

```
公式評価ハーネス:
- 存在: YES / NO
- コマンド: [確認したコマンド]
- 使用判断: 必須 / 推奨 / 不要
```

#### 3. Docker要件確認

```
Docker要件:
- Dockerfile: YES / NO
- docker-compose.yml: YES / NO
- Docker必須: YES / NO
```

#### 4. 実装方針提案

```
提案する実装方針:

Option 1: 公式ハーネス使用（推奨）
  - メリット: 標準準拠、比較可能
  - デメリット: [あれば記載]
  - 採用判断: ✅推奨

Option 2: Docker統合
  - メリット: [記載]
  - デメリット: [記載]
  - 採用判断: [記載]

Option 3: 独自実装
  - メリット: [記載]
  - デメリット: 標準非準拠
  - 採用判断: ❌非推奨

どの方針で進めますか？
```

### Phase 3: ユーザー承認待ち

**ユーザー様の承認を得るまで実装を開始しないでください。**

承認を得た後、以下を実行：

```bash
# チェックリストの該当項目にチェックを入れる
# 実装を開始する
```

---

## 📝 記録

実装開始前に以下を記録：

- 調査実施日時: [YYYY-MM-DD HH:MM]
- 公式リポジトリURL: [URL]
- 公式ハーネス: [有無]
- Docker要件: [有無]
- 採用方針: [Option番号]
- ユーザー承認: ✅/❌

---

## ⚠️ 注意事項

1. このチェックを**スキップしない**
2. ユーザー承認を**勝手に省略しない**
3. 「速度優先」を理由に**ショートカットしない**
